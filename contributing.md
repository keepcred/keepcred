## Политика безопасности
1. **Если Вы нашли дыру в безопасности, НЕ СОЗДАВАЙТЕ публичный вопрос (issue) и пишите на почту n1k@the-dev.ru с темой `keepcred: security error - <Ваш текст (опционально)>`;**

## Коммиты
1. Код должен быть отформатирован в соттветсвии с конфигурацией `prettier.config.ts` проекта;
2. Код должен быть проверен с помощью линтера `eslint.config.ts` проекта;
3. Код должен быть проверен с помощью `typescript` на отсутствие ошибок типизации;

## Вопросы (issues)
1. Для внесения предложений/пожеланий по проекту, необходимо использовать заготовленную форму для формирования предложения, которая должна включать в себя нумерованный список от 1 до n предложений;
2. Для информирования об ошибках/багах в работе приложения, которые не имеют отношения к [политике безопасности](#политика-безопасности), необходимо использовать заготовленную форму для формирования отчёта об обнаруженной ошибке с информацией требуемой формой;
